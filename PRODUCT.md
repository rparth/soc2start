# Product

## Register

product

## Users

CISOs and compliance leaders at organizations pursuing or maintaining SOC2 certification. They may not be deeply technical but need tools that don't require training to use. They work under audit deadlines, manage cross-functional evidence collection, and report to leadership on compliance posture. Intuitive, self-explanatory workflows are non-negotiable.

## Product Purpose

SOC2Start.io is a self-hostable GRC platform that makes compliance work feel like engineering work. It covers risk management, control tracking, vendor risk, data privacy, access reviews, audit programs, and document workflows. Success looks like: a CISO can go from zero to audit-ready without ever feeling lost or needing to read documentation.

## Brand Personality

Expert, calm, precise. Like a seasoned security advisor who makes compliance feel manageable, not intimidating. Confident without being corporate. Technical without being cold.

Three words: **assured, clear, purposeful**.

## Anti-references

- **Legacy GRC platforms** (ServiceNow, OneTrust, Vanta's older UI): cluttered enterprise dashboards, overwhelming form density, grey-on-grey monotony, tiny text, modal-on-modal workflows.
- **Generic SaaS cream aesthetic**: the 2024 template of soft beige gradients, rounded-everything, indistinct branding. SOC2Start.io has a distinct identity (navy + emerald), not a theme kit.
- **Overly playful tools**: emoji-heavy interfaces, whimsical illustrations, pastel palettes. Compliance demands credibility; playfulness undermines it.

## Design Principles

1. **Show the path**: every screen answers "what do I do next?" Compliance is complex; the UI absorbs that complexity so the user doesn't have to.
2. **Earn trust through precision**: pixel-level consistency, correct terminology, zero ambiguity. A compliance tool that looks sloppy is a compliance tool nobody trusts.
3. **Density with clarity**: show enough information to be useful without overwhelming. Dense when scanning (tables, lists), spacious when acting (forms, approvals).
4. **Intuitive by default**: every interaction should be self-explanatory. No training required, no documentation dependency. If a user has to ask how something works, the UI failed.

## Accessibility & Inclusion

WCAG 2.1 AA minimum. All interactive elements keyboard-accessible. Color is never the sole indicator of state. Respect prefers-reduced-motion. Support system dark mode.
